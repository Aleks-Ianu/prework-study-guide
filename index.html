Git

  • git status: checks what branch we are currently on

  • git checkout -b branc-name: creates a new branch and switches to it

  • Make sure we are on the feature/add-html branch from our command line. We have to add the changed files in our working directory to the staging area of Git. Use the following command in your terminal to stage your changes:git add -A

  • Now that all of our changes have been staged, we can commit them to our repository. Use the following command in your terminal to commit the changes: git commit -m "added code to HTML file"

  • Next, let's make sure that our current branch is up to date before opening a pull request. We need to sync our local branch with the remote branch in GitHub. This is especially important in a team environment in case a team member has added a new feature to the project. The process of syncing and merging code into the GitHub repo is how the project gets continually updated with more features and content from different team members. Use the following command in your terminal to pull in the base branch: git pull origin main

  • Let's push up all of our changes by using the following command: git push origin feature/add-html/li>
